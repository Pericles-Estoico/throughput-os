# Story 1.1 — Sales Intelligence: Importação de Vendas e Cálculo de Ganho

## Status
Ready for Review

## Story
**Como** usuário do Throughput OS,
**Quero** importar dados de vendas por SKU e ver o ganho (throughput) calculado automaticamente,
**Para** saber quais produtos realmente geram dinheiro e tomar decisões baseadas em ganho real.

## Acceptance Criteria

1. **Tabela de vendas** exibe: SKU, Canal, Quantidade, Preço Médio, Receita Líquida, Ganho Unitário, Ganho Total
2. **Formulário de importação** permite inserir venda manual com todos os campos da tabela `sales`
3. **Ganho unitário** calculado automaticamente: `net_revenue - variable_cost_total`
4. **Ganho total** calculado: `throughput_unit × quantity`
5. **Ranking automático** por ganho total (maior → menor)
6. **Filtros** funcionais: período (data), canal, categoria
7. **Estados vazios** exibidos quando não há dados
8. **Validação** de campos obrigatórios no formulário

**Schema Supabase (tabelas envolvidas):**
- `sales`: id, sku_id, date, channel, quantity, unit_price, gross_revenue, fees, freight, tax, net_revenue (computed)
- `costs`: sku_id, variable_cost_total (computed)
- `skus`: id, sku_code, product_name, category, family, active

**Query base para métricas:**
```sql
SELECT
  s.id, s.date, s.channel, s.quantity, s.unit_price,
  s.net_revenue,
  sk.sku_code, sk.product_name, sk.category,
  c.variable_cost_total,
  (s.net_revenue - c.variable_cost_total) as throughput_unit,
  (s.net_revenue - c.variable_cost_total) * s.quantity as throughput_total
FROM sales s
JOIN skus sk ON sk.id = s.sku_id
LEFT JOIN costs c ON c.sku_id = s.sku_id
ORDER BY throughput_total DESC
```
