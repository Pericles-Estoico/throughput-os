# Story 1.3 — Execution Dashboard: O Que Fazer Hoje

## Status
Ready for Dev

## Story
**Como** usuário do Throughput OS,
**Quero** abrir o sistema e ver imediatamente o que devo fazer hoje,
**Para** tomar decisões sem precisar analisar dados manualmente.

## Acceptance Criteria

1. **4 cards de KPIs** exibem dados reais: Ganho do Dia, SKUs A vendidos hoje, SKUs em ruptura, Gargalo atual
2. **Ação crítica do dia** gerada automaticamente com base nos alertas ativos
3. **5 tipos de alertas** detectados automaticamente:
   - Produto A em ruptura (cobertura < 3 dias)
   - Produto D consumindo produção
   - Gargalo parado (sem vendas do dia)
   - Estoque acima de 20 dias
   - Venda sem margem (throughput negativo)
4. **Lista de alertas** exibida com prioridade (crítico primeiro)
5. **Dashboard atualiza** os dados a cada acesso (sem cache estático)

## Dev Notes

**Tipo DailyAlert já definido em `src/types/index.ts`:**
```typescript
interface DailyAlert {
  type: AlertType
  sku_id?: string
  sku_name?: string
  message: string
  priority: 'critical' | 'warning' | 'info'
}
```

**Regras de alerta (do PRD):**
- ruptura: cobertura < 3 dias → critical
- produto_ruim: SKU D com estoque → warning
- excesso: cobertura > 20 dias → warning
- sem_margem: throughput < 0 → critical
- gargalo: sem produção registrada → warning
